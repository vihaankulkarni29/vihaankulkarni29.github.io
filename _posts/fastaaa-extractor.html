FastaAAExtractor

Status: Active Development
Year: 2024
Type: Automation Tool

Automated amino acid extraction from unannotated genomic data, reducing manual curation by over 90%.

View on GitHub → Installation Guide

Overview

FastaAAExtractor was born out of a frustrating reality in computational genomics: researchers were spending 40+ hours manually extracting and curating amino acid sequences from unannotated genomic data. What should be a simple preprocessing step was becoming a major bottleneck in comparative genomics workflows.

This tool automates the entire extraction process, handling batch processing of FASTA files while maintaining the accuracy needed for downstream analysis. It's particularly valuable for AMR research where identifying specific protein variants across hundreds or thousands of isolates is routine work.

The Problem

In antimicrobial resistance research, scientists frequently need to extract specific amino acid sequences from bacterial genomes to study protein variants. Traditional approaches involve:

  • Manually identifying target genes in unannotated assemblies
  • Extracting sequences one file at a time
  • Dealing with inconsistent file formats and naming conventions
  • Validating extraction accuracy through visual inspection
  • Reformatting output for downstream tools

For a typical study with 500 isolates, this process could take 2-3 weeks of full-time work. More critically, the manual nature introduced errors and made reproduction difficult.

Key Insight: The bottleneck wasn't computational power—it was the human time required to supervise each extraction step and handle edge cases in messy real-world genomic data.

The Solution

FastaAAExtractor provides a robust, automated pipeline that handles the entire extraction workflow:

Core Features

  • Batch Processing: Process hundreds of FASTA files with a single command
  • Flexible Targeting: Support for multiple extraction strategies (header matching, sequence similarity, position-based)
  • Error Handling: Graceful handling of malformed files, missing sequences, and ambiguous matches
  • Quality Control: Automated validation checks and detailed logging
  • Format Flexibility: Handles various FASTA formats and outputs in researcher-specified formats

Technical Approach

The tool is built on Biopython's SeqIO module but adds significant functionality for production use:

  • Custom parsers for handling common genomic data quirks
  • Parallel processing for large datasets
  • Comprehensive logging system for troubleshooting
  • Configurable parameters through YAML config files

Impact: What previously took 40 hours now completes in under 4 hours, with higher accuracy and complete reproducibility.

Installation & Usage

Prerequisites

Python 3.8+ Biopython NumPy PyYAML

Installation

# Clone the repository
git clone https://github.com/vihaankulkarni29/FastaAAExtractor.git
cd FastaAAExtractor

# Install dependencies
pip install -r requirements.txt

# Run tests (optional)
python -m pytest tests/

Basic Usage

# Extract sequences using default settings
python extract.py --input genomes/ --output results/ --config config.yaml

# Process with custom parameters
python extract.py --input genomes/ --output results/ \
  --target "resistance_gene" --format clustal

Configuration Example

# config.yaml
extraction:
  method: "header_match"  # or "similarity" or "position"
  target_pattern: ".*resistance.*"
  min_length: 300
  max_length: 1500
  
output:
  format: "fasta"  # or "clustal", "phylip"
  naming: "isolate_gene"
  
quality:
  validate: true
  min_coverage: 0.95

Lessons Learned

What Worked Well

  • Focus on Real-World Data: Testing with messy, real genomic data from day one meant the tool handled edge cases that broke other solutions
  • Comprehensive Logging: Detailed logs made troubleshooting trivial and built user trust
  • Flexible Configuration: YAML configs made it easy for non-programmers to customize behavior

Challenges

  • Format Variations: FASTA "standards" are more like suggestions—handling all variations required extensive testing
  • Performance Optimization: Initial version was too slow for 1000+ isolates; required adding parallel processing
  • Error Communication: Making error messages actionable for biologists (not programmers) took multiple iterations

Future Improvements

  • Add GUI for researchers who prefer not to use command line
  • Integration with common genome annotation tools
  • Support for cloud storage (S3, GCS) for large datasets
  • Machine learning-based sequence targeting for ambiguous cases
